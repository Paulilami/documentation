## Core Functions

### Supply Management

**Minting**: Issue new tokens to specified addresses (if enabled)
- Automatic compliance checks before minting
- Optional fee collection based on token configuration
- Supply limit enforcement

**Burning**: Permanently remove tokens from circulation
- Reduces total supply
- Updates on-chain registries
- Maintains audit trail

### Account Controls

**Freeze/Unfreeze**: Temporarily halt token transfers for specific addresses
- Instant enforcement across all operations
- Maintains token balance while frozen
- Reversible through API call

**Whitelist Management**: Control who can hold and transfer tokens
- Add or remove addresses in bulk
- Automatic transfer validation
- Works with KYC verification

### Token Information

**Real-time Data**: Query current token state and configuration
- Total supply and burned amounts
- Holder counts and distributions
- Active restrictions and limits

## Compliance Enforcement

All management operations automatically enforce the compliance rules set during token creation:

- **KYC Requirements**: Transfers blocked for non-verified addresses
- **Whitelist Restrictions**: Only approved addresses can receive tokens
- **Balance Limits**: Maximum holdings per wallet enforced
- **Monthly Limits**: Transfer volume restrictions per period

## API Endpoints

### Token Operations
- `POST /api/token/{tokenAddress}/mint` - Issue new tokens
- `POST /api/token/{tokenAddress}/burn` - Burn tokens
- `GET /api/token/{tokenAddress}/info` - Get token details
- `GET /api/tokens` - List all managed tokens

### Compliance Controls
- `POST /api/token/{tokenAddress}/freeze` - Freeze account
- `POST /api/token/{tokenAddress}/unfreeze` - Unfreeze account
- `POST /api/token/{tokenAddress}/whitelist` - Manage whitelist

## Integration Pattern

```javascript
// Example: Mint tokens with compliance check
const mintTokens = async (tokenAddress, recipient, amount) => {
  const response = await fetch(`/api/token/${tokenAddress}/mint`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'x-api-key': 'YOUR_INSTANCE_API_KEY'
    },
    body: JSON.stringify({
      to: recipient,
      amount: amount
    })
  });
  
  return response.json();
};
```

## Webhook Events
Monitor token operations in real-time through webhook notifications:

- **token_minted** - New tokens issued
- **token_burned** - Tokens removed from circulation
- **account_frozen** - Account restricted
- **whitelist_updated** - Approved addresses changed

<Note>
All management functions are available only to authorized API keys. Token creators maintain full control while authority roles handle compliance operations.
</Note>
