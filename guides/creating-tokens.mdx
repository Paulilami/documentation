# Creating Security Tokens

The Trusset platform provides multiple ways to create security tokens, from simple standard tokens to fully customized assets. This guide walks you through the token creation process.

## Token Creation Methods

<Tabs>
  <Tab title="Standard Token">
    Create basic tokens with minimal configuration. Best for beginners and simple use cases.
    
    ```typescript
    createStandardAsset(
      name,            // Token name (e.g., "My Company Shares")
      symbol,          // Token symbol (e.g., "MCS")
      initialSupply,   // Initial number of tokens to mint
      headerImage,     // Image URL for the token
      assetType        // Type of asset (EQUITY, BOND, etc.)
    )
    ```
  </Tab>
  
  <Tab title="Decentralized Token">
    Create tokens with decentralized governance and management features.
    
    ```typescript
    createDecentralizedAsset(
      name,            // Token name
      symbol,          // Token symbol
      initialSupply,   // Initial number of tokens
      headerImage,     // Image URL
      maxSupply,       // Maximum possible supply
      authority,       // Optional authority address
      assetType        // Type of asset
    )
    ```
  </Tab>
  
  <Tab title="Compliance Token">
    Create tokens with built-in compliance features like KYC, whitelisting, and more.
    
    ```typescript
    createComplianceAsset(
      name,            // Token name
      symbol,          // Token symbol
      initialSupply,   // Initial number of tokens
      headerImage,     // Image URL
      maxSupply,       // Maximum possible supply
      authority,       // Authority address
      enableKYC,       // Whether KYC is required
      enableBlacklist, // Whether blacklisting is enabled
      enableWhitelist, // Whether whitelisting is enabled
      enableFreeze,    // Whether account freezing is enabled
      assetType        // Type of asset
    )
    ```
  </Tab>
  
  <Tab title="Custom Token">
    Full control over all token parameters and features.
    
    ```typescript
    createFullyConfiguredAsset(
      name,                 // Token name
      symbol,               // Token symbol
      initialSupply,        // Initial number of tokens
      headerImage,          // Image URL
      flags,                // Capability flags (see Learn section)
      maxSupply,            // Maximum possible supply
      feeBps,               // Fees in basis points (1/100 of 1%)
      authority,            // Authority address
      feeReceiver,          // Address receiving fees
      assetType,            // Type of asset
      monthlyTransferLimit, // Max transfers per month
      maxPerWallet,         // Max tokens per wallet
      fractions             // Fractional units (0-10)
    )
    ```
  </Tab>
</Tabs>

## Creating a Token in the UI

1. Navigate to the **Token Studio** section
2. Select the token type you want to create
3. Fill in the required fields:
   - **Token Name**: Full name of your token
   - **Token Symbol**: Short identifier (2-5 characters recommended)
   - **Initial Supply**: Number of tokens to mint initially
   - **Asset Type**: Category of the security token
   - **Asset Image**: IPFS URL created through a Pinata gateway, allowing users to simply upload their images. You can leave this field empty (not recomended)

4. Configure additional settings based on token type:
   - Compliance requirements
   - Transfer restrictions
   - Governance parameters

5. Click **Create Token**
6. Confirm the transaction in your wallet

<Note>
All tokens are created on the Sepolia testnet. Transaction costs are paid in Sepolia ETH.
</Note>

## Asset Types

The platform supports multiple security token types:

- **EQUITY**: Represents ownership in a company
- **BONDS**: Represents debt instruments
- **STOCKS**: Traditional company shares
- **REAL_ESTATE**: Property-backed tokens
- **CASH_EQUIVALENT**: Stable value instruments
- **DEBT**: General debt instruments
- **OTHER**: Custom asset types

## Token Parameters Explained

- **Name**: The full name of your token (e.g., "Company A Shares")
- **Symbol**: Short identifier for your token (e.g., "CAS")
- **Initial Supply**: Number of tokens created at launch
- **Max Supply**: Maximum possible supply (can equal initial supply for fixed supply)
- **Asset Image**: Visual representation of your collection of tokens
- **Flags**: Configuration options for token capabilities (see [Token Flags](/learn/token-flags))
- **Fee BPS**: Fee amounts in basis points for transactions, if enabled
- **Authority**: Address with special permissions to manage the token
- **Fee Receiver**: Address that collects any fees from token operations
