## Overview

The Identity & KYC API provides institutional-grade identity verification infrastructure for tokenized assets. This system enables issuers to verify user identities, manage KYC status, and enforce compliance requirements directly on-chain.

The system provides:

1. **Batch Verification** - Process hundreds of KYC verifications in a single transaction
2. **Country Compliance** - Store country codes for jurisdiction-specific requirements
3. **Expiry Management** - Set automatic expiration dates for time-limited verifications
4. **Bulk Operations** - Update or revoke multiple identities efficiently
5. **On-chain Storage** - Permanent, auditable record of all verifications

## Verification Flow

1. **Collect KYC Data** - Gather user addresses and country information
2. **Batch Verify** - Submit verification data via CSV or JSON
3. **Monitor Status** - Track batch processing progress
4. **Update as Needed** - Modify or revoke verifications when required
5. **Query Status** - Check individual user verification status

## Key Features

### Batch Processing
- **CSV Upload**: Process KYC data from CSV files with automatic parsing
- **JSON Support**: Submit structured JSON data for verification
- **Chunked Processing**: Automatic chunking for large batches (500+ records)
- **Progress Tracking**: Real-time batch processing status

### Data Format Support
- **Flexible Headers**: Recognizes common address and country field names
- **Auto-validation**: Validates Ethereum addresses and country codes
- **Error Handling**: Reports invalid records without failing entire batch

### Compliance Features
- **Country Codes**: ISO 3166-1 alpha-2 country code storage
- **Expiry Dates**: Optional Unix timestamp expiry for time-limited KYC
- **Issuer Attribution**: Each verification linked to issuer address
- **Audit Trail**: Complete history of verifications and revocations

### Gas Optimization
- **Batch Operations**: Reduces gas costs by 80% compared to individual transactions
- **Automatic Chunking**: Splits large batches to avoid gas limits
- **Retry Logic**: Automatic retry with exponential backoff for failed transactions
