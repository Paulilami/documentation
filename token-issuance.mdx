## Token Features

### Compliance Controls
- **KYC Verification**: Automatic checks for recipient wallet compliance
- **Whitelist/Blacklist**: Control who can hold and transfer tokens
- **Transfer Restrictions**: Set maximum holdings per wallet and monthly limits
- **Freeze Functionality**: Temporarily halt specific accounts

### Token Operations
- **Minting**: Issue new tokens to specified addresses
- **Burning**: Permanently remove tokens from circulation
- **Metadata**: Attach documents and tags to tokens
- **Fee Collection**: Optional fees for minting, burning, or transfers

## Creation Process

1. Configure token parameters (name, symbol, supply, compliance rules)
2. Specify recipient wallet address
3. System checks recipient KYC status automatically
4. Token deploys on-chain with all rules embedded
5. Receive token contract address for management

## API Endpoints

- `POST /api/issuance/tokens` - Deploy new security tokens
- `GET /api/issuance/tokens` - List all tokens created by your instance
- `PATCH /api/issuance/tokens/{id}` - Track deployment progress

<Note>
All tokens are ERC1400 compliant and include transfer restrictions by default. KYC verification is checked automatically for recipient addresses.
</Note>
